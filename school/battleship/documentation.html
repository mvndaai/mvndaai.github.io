Grading Requirements


10 Documentation explaining how you implemented your game and how it satisfies the remainder of the requirements. The documentation should be in a file named documentation.html, with a link to the documentation file on the project description page.
This page is the documentation. This table tells how each of the requiremnts are implemented.
20 Game grid that uses CSS and is generated by JavaScript
The gameboards are tabled generated by javascript, using CSS for the colors and sizing.
20 User input in the form of text, select options, and/or buttons
The user can select game size by select options. They can load the internal save by a button, and can reset at any time by buttons.
20 User input in the form of mouse clicks
The user can attack the computer by clicking on the grid.
20 Dynamic modification of HTML using innerHTML or DOM functions
The gameboards are added dynamically as well as ships,hits and misses using innerHTML.
20 Use of XMLHttpRequest to load JSON or XML game data that is displayed on the grid page
The game can be load a demo save using XMLHttpRequest by clicking the "Load Demo State" button.
20 Incorporates HTML 5 audio tags, video tags, canvas, or local storage
The game uses localStorage. The game state is saved after every attack and will automatically load the current state when reopening/refreshing the page. It keeps a count of wins for user vs computer over time.
120 Game logic code
The computer attacks around a hit ship until it finds two next to each other, then follows that row or column until it misses in both directions. It also displays numbers of ships left as well as ending when someone won. The ships are safely placed randomly on the grid.
250 TOTAL
